# コア・ユーザー・ジャーニー (Core User Journey)

## 1. Project Initiation (プロジェクト開始)
**「ふわっとした要件から、具体的なタスクへ」**

* **User Action:**
    * マネージャーが「新規プロジェクト作成」画面で、プロジェクト名と「ざっくりとした目的（Mission）」を入力する。
    * 例：「来月の展示会用に、サービスのLPをリニューアルしたい」
* **System Action (AI):**
    * AIが目的を解析し、必要なフェーズ（要件定義、デザイン、実装...）と具体的なタスク（WBS）を自動生成する。
    * 各タスクに適した役割（Frontend, Designerなど）と想定工数を提案する。
* **Success:**
    * 数分でプロジェクトの計画（ドラフト）が完了し、即座に開始できる。

## 2. Daily Reporting (日々の報告)
**「帰り際の1分で、完璧な日報を」**

* **User Action:**
    * メンバーは退勤前に、その日やったことを箇条書きで入力する。
    * 気分や状況に合わせて「AI補正レベル」スライダーを調整する（例：通常はLv3、ミスした日はLv5【謝罪モード】）。
* **System Action (AI):**
    * **清書:** 箇条書きを、ビジネス枕詞（「お疲れ様です」等）を含んだ完璧なビジネス文書に変換する。
    * **工数抽出:** 文章の内容から「どのタスクに何時間使ったか」を推論し、バックグラウンドで工数ログを記録する。
* **Success:**
    * ユーザーは「文章を考える」苦痛から解放される。
    * マネージャーは「工数入力して」と催促するストレスから解放される。

## 3. Weekly Review (週次の振り返り)
**「金曜日の憂鬱を、達成感へ」**

* **User Action:**
    * 金曜日の夕方、マネージャー（またはメンバー）が「週報作成」ボタンを押す。
    * 対象期間（今週）を指定する。
* **System Action (AI):**
    * チームメンバー全員の1週間分の日報と、蓄積された工数データを読み込む。
    * 「今週のハイライト」「進捗状況」「課題」「来週の予定」を構造化して要約する。
* **Success:**
    * 数秒で、そのまま部長報告や定例会で使える「週報」が完成する。
    * 集計作業やコピペ作業がゼロになる。

## 4. Feedback & Cycle (評価と改善)
**「データに基づく健全な運営」**

* **User Action:**
    * マネージャーはダッシュボードで、プロジェクトの予実（予定工数 vs 実績工数）を確認する。
* **Success:**
    * 「誰が何に時間を使っているか」が可視化され、感覚ではなくデータに基づいたリソース調整が可能になる。
    * メンバーは「報告業務」ではなく「開発成果」で正当に評価される。
