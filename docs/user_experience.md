# ユーザー体験 (User Experience)

## プロダクトコンセプト
**「事実は人間が、建前はAIが」**

AI Project Governorは、日本企業（JTC）特有の「報告業務」にかかる精神的・時間的コスト（Toil）を徹底的に排除し、エンジニアが本質的な価値創造に集中できる環境を作ります。

## ターゲットペルソナ

### 1. ユーザー（メンバー層）
* **属性:** 開発者、エンジニア、現場担当者。
* **課題:**
    * 開発作業は好きだが、日報や週報を書くのが苦痛。
    * 上司の顔色を伺った「角が立たない表現」を考えるのに時間がかかる。
    * 「工数入力」を頻繁に忘れて怒られる。
* **求める体験:**
    * 帰り際にチャット感覚で「事実」を投げるだけで帰りたい。
    * 面倒な事務作業はシステムに勝手にやってほしい。

### 2. 管理者（マネージャー層）
* **属性:** プロジェクトマネージャー、リーダー、管理職。
* **課題:**
    * メンバーに進捗確認や工数入力を催促したくない（マイクロマネジメントと思われたくない）。
    * チームの状況を定量的に把握したいが、データが集まらない。
    * 部長や役員への報告資料作成（メンバーの報告のまとめ）に金曜日の夜を費やしている。
* **求める体験:**
    * 何も言わなくても、正確な進捗と工数データが集まっている状態。
    * 上位層への報告用サマリが自動で生成されること。

## コア・バリュー (UX Principles)

1.  **Stress-free Input (ストレスフリーな入力)**
    * ユーザーは構成や丁寧語を一切気にする必要がない。「事実の箇条書き」だけで良い。
    * TPO（謝罪、役員報告など）に合わせた文体変換はAIが担う。

2.  **Invisible Governance (透明な管理)**
    * 「工数管理」という作業をユーザーから隠蔽する。
    * 日報のテキストからAIがタスクと時間を推論し、裏側で勝手に集計する。

3.  **High-Quality Output (評価されるアウトプット)**
    * AIが生成する文章は、そのまま「お堅い企業」の上司に提出しても通用するレベルを保証する。
    * 個人の評価やチームの信頼につながる品質を担保する。
